------------------------------------------------------------
A. ΚΑΤΗΓΟΡΙΕΣ ΧΡΗΣΤΩΝ (Roles)
------------------------------------------------------------

✅ A1. Student - Cook (Provider): Υπάρχει, ο χρήστης μπορεί να δημιουργεί αγγελίες.

✅ A2. Student - Consumer (Receiver): Υπάρχει, ο χρήστης μπορεί να αιτείται μερίδες.

✅ A3. Administrator (Admin): Υπάρχει. Ο admin έχει AdminDashboard με στατιστικά
   και leaderboard. Ο ρόλος ελέγχεται μέσω req.session.role.

------------------------------------------------------------
B. COOK FUNCTIONS
------------------------------------------------------------

⚠️  B1. Listing Management (CRUD):
   - CREATE: ✅ Υπάρχει (CreatePostForm.js + POST /api/posts)
   - EDIT: ❌ ΛΕΙΠΕΙ από το UI. Υπάρχει το endpoint PUT /api/posts/:id στον server
     αλλά δεν υπάρχει κανένα κουμπί Edit στο Dashboard ή αλλού για να επεξεργαστεί
     ο cook την αγγελία του.
   - DELETE: ❌ ΛΕΙΠΕΙ από το UI. Υπάρχει DELETE /api/posts/:id αλλά δεν υπάρχει
     κουμπί διαγραφής στο Dashboard.
   - Φωτογραφία (optional): ✅ Υλοποιήθηκε πρόσφατα με upload.
   - Αλλεργιογόνα: ✅ Υπάρχουν ως text field στη φόρμα.
   - 48h auto-delete: ✅ Υπάρχει στο GET /api/posts (φιλτράρει posts > 48h).
     ΟΜΩΣ δεν γίνεται αυτόματο "marking as deleted" - απλά δεν εμφανίζονται.
     Η εκφώνηση λέει να σημαίνονται ως deleted στη βάση. ΜΕΡΙΚΩΣ υλοποιημένο.

✅ B2. Delivery Details: Το pickup_location και pickup_time είναι υποχρεωτικά
   στη φόρμα. Υπάρχει και χάρτης Leaflet για επιλογή τοποθεσίας.

✅ B3. Request Management:
   - Αποδοχή/Απόρριψη: ✅ Υπάρχουν κουμπιά στο Dashboard (Αποδοχή/Απόρριψη).
   - Αυτόματη μείωση μερίδων κατά αποδοχή: ✅ Στον server (completion route).
   - "Παραδόθηκε": ✅ Κουμπί στο Dashboard.
   - "Δεν Εμφανίστηκε" (no-show): ✅ Κουμπί στο Dashboard + αφαίρεση 1 credit
     από τον consumer στον server (server.js:576-580).

✅ B4. Points Display: Τα credits φαίνονται στο Dashboard header.
   Λογική: αξιολόγηση > 3 → +2 credits, ≤ 3 → +1 credit. ✅

------------------------------------------------------------
C. CONSUMER FUNCTIONS
------------------------------------------------------------

⚠️  C1. Dynamic Feed:
   - Λίστα αγγελιών: ✅ Υπάρχει (Feed.js).
   - Sold-out highlighting: ✅ Φιλτράρεται client-side (available_portions > 0).
   - Εμφάνιση σε ΧΑΡΤΗ: ❌ ΛΕΙΠΕΙ. Το Feed δεν έχει map view.
     Η εκφώνηση λέει ρητά "Display... on a map". Ο Leaflet υπάρχει μόνο
     στη φόρμα δημιουργίας αγγελίας.
   - Φιλτράρισμα/ταξινόμηση βάσει απόστασης: ❌ ΛΕΙΠΕΙ. Υπάρχει endpoint
     GET /api/posts/search με sort parameter αλλά δεν χρησιμοποιείται στο Feed.
     Δεν υπάρχει "sort by distance from location" στο UI.

✅ C2. Sending a Request:
   - Κουμπί αίτησης μερίδας: ✅ Υπάρχει στο Feed.
   - Έλεγχος credits (τουλάχιστον 1): ✅ Γίνεται στον server (server.js:419).
   - 5 credits στους νέους χρήστες: ✅ Default στη βάση (users.credits = 5).

⚠️  C3. Rating System:
   - 1-5 αστέρια: ✅ Υπάρχει (select dropdown στο Dashboard).
   - Εμφατική πρόταση αξιολόγησης: ✅ Υλοποιήθηκε πρόσφατα (banner + toast).
   - Αφαίρεση 1 credit αν δεν αξιολογήσει εντός 48 ωρών: ❌ ΛΕΙΠΕΙ.
     Η εκφώνηση (C3) λέει ρητά ότι αν κάποιος δεν αξιολογήσει εντός 48 ωρών
     αφαιρείται 1 credit. Δεν υπάρχει cron job ή scheduler που να το κάνει αυτό.

------------------------------------------------------------
D. ADMIN FUNCTIONS
------------------------------------------------------------

✅ D1. Overall Statistics: Υπάρχει AdminDashboard με στατιστικά (σύνολο μερίδων
   τελευταίου μήνα μέσω GET /api/admin/stats).

✅ D2. Leaderboard: Υπάρχει "Top Donor" και στατιστικά (server.js:784).

------------------------------------------------------------
E. TECHNICAL REQUIREMENTS
------------------------------------------------------------

✅ E1. DOM & Events: Χρησιμοποιείται addEventListener, e.preventDefault(),
   δυναμική ενημέρωση χωρίς reload (SPA αρχιτεκτονική).

✅ E2. Client-Server Communication: fetch() API + JSON παντού.

⚠️  E3. Responsive Design: Υπάρχουν media queries στο style.css αλλά η εφαρμογή
   δεν έχει ελεγχθεί πλήρως σε mobile. Το sidebar/nav χρειάζεται έλεγχο.

------------------------------------------------------------
ΣΥΝΟΨΗ ΕΛΛΕΙΨΕΩΝ (ΚΡΙΣΙΜΕΣ)
------------------------------------------------------------

1. ❌ EDIT/DELETE αγγελίας στο UI (το backend υπάρχει, λείπει το frontend).

2. ❌ ΧΑΡΤΗΣ στο Feed - δεν υπάρχει map view με pins για τα διαθέσιμα φαγητά.

3. ❌ Ταξινόμηση/φιλτράρισμα βάσει ΑΠΟΣΤΑΣΗΣ στο Feed.

4. ❌ Αυτόματη αφαίρεση 1 credit αν ο consumer δεν αξιολογήσει εντός 48 ωρών
   (χρειάζεται cron job / scheduled task στον server).

5. ⚠️  Αυτόματο "marking as deleted" μετά 48 ώρες - τώρα γίνεται φιλτράρισμα
   κατά την εμφάνιση αλλά δεν ενημερώνεται το status στη βάση.
